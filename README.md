# Photo Vote Wedding — инструкция по запуску

Инструкция написана для человека без опыта программирования. Каждый шаг — это клики в браузере, копирование и вставка. Понадобится ~30-40 минут один раз.

## Что вам нужно перед началом
- Аккаунт на supabase.com (бесплатно)
- Аккаунт на vercel.com (бесплатно)
- Аккаунт на github.com (бесплатно) — туда мы загрузим код

---

## Шаг 1. Создать проект в Supabase

1. Зайдите на supabase.com → New Project.
2. Придумайте имя проекта и надёжный пароль базы данных (сохраните его куда-нибудь).
3. Дождитесь, пока проект создастся (1-2 минуты).
4. Слева откройте SQL Editor → New query.
5. Откройте файл supabase/schema.sql из этого проекта, скопируйте весь текст и вставьте в редактор.
6. Нажмите Run. Должно появиться "Success. No rows returned".

## Шаг 2. Включить анонимный вход гостей

1. В Supabase зайдите в Authentication → Sign In / Providers.
2. Найдите Anonymous Sign-Ins и включите переключатель.

## Шаг 3. Создать хранилище для фото

1. Слева откройте Storage → New bucket.
2. Имя бакета: wedding-photos (именно так, с дефисом).
3. Включите Public bucket.
4. Создайте.

## Шаг 4. Забрать ключи проекта

1. Слева Project Settings → API.
2. Скопируйте Project URL и anon public key — они понадобятся на шаге 6.

## Шаг 5. Создать первое событие (свадьбу)

Снова SQL Editor → New query, выполните (замените текст на свои данные):

```sql
insert into events (code, title) values ('SOFIA-JAMES-2026', 'София и Джеймс');
```

code — часть будущей ссылки, без пробелов и кириллицы, например SOFIA-JAMES-2026.

## Шаг 6. Настроить и запустить сайт

1. Скачайте папку проекта себе на компьютер.
2. Переименуйте файл .env.example в .env.local и впишите туда значения из шага 4:

```
VITE_SUPABASE_URL=вставьте сюда Project URL
VITE_SUPABASE_ANON_KEY=вставьте сюда anon public key
```

3. Загрузите папку проекта в новый репозиторий на GitHub (можно через github.com → "Add file" → "Upload files", без командной строки).
4. Зайдите на vercel.com → Add New Project → выберите этот репозиторий.
5. В настройках проекта на Vercel добавьте те же две переменные окружения (VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY) — раздел Environment Variables.
6. Нажмите Deploy. Через минуту сайт будет готов, Vercel даст вам ссылку вида https://ваш-проект.vercel.app.

## Шаг 7. Стать организатором

1. Откройте https://ваш-сайт/admin/SOFIA-JAMES-2026 (подставьте свой код события).
2. Нажмите "Создать аккаунт", введите email и пароль — это ваш личный кабинет организатора.
3. В Supabase зайдите в Authentication → Users, скопируйте UID только что созданного пользователя.
4. В SQL Editor выполните (подставив свои UID и id события — id события можно увидеть в таблице events в разделе Table Editor):

```sql
insert into admins (user_id, event_id)
  values ('вставьте сюда UID пользователя', 'вставьте сюда id события');
```

5. Обновите страницу /admin/SOFIA-JAMES-2026 — откроется кабинет организатора.

## Шаг 8. Распечатать QR-коды

Откройте https://ваш-сайт/admin/SOFIA-JAMES-2026/qr, скачайте PNG и распечатайте на карточках столов.

## Шаг 9. В конце вечера

В кабинете организатора (/admin/SOFIA-JAMES-2026) нажмите «Объявить победителем» у фото-лидера. Экран для показа на проекторе/экране: /event/SOFIA-JAMES-2026/winner.

---

## Важные ограничения этой версии (чтобы не было сюрпризов)

- Один гость = один браузер. Если гость очистит кэш или откроет сайт в режиме инкогнито, приложение попросит представиться заново — это создаст нового "гостя" с тем же именем. Для дружеского конкурса это не критично.
- Один аккаунт сайта = одна свадьба. Для второго такого конкурса на другом мероприятии разверните отдельную копию сайта (повторите шаги 6-8).
- Фото публикуются сразу, без предварительной проверки. Организатор может в любой момент скрыть или удалить неудачное фото из кабинета — но до этого момента оно видно всем гостям.
- Хранилище фото публичное по прямой ссылке на файл — но сам адрес узнать без доступа к панели нельзя, а страница события не отображается в поисковиках.
